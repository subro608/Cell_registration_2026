Technique to isolate the central hippocampal region from nd2 confocal tiles, masking out the outer ring tissue and borders.

**Why:** Prof's directive — "ignore the peripheral sections outside center circle structure" when stitching tiles. Only the central hippocampal island matters for alignment and registration.

**How to apply:** Use per-tile VIA masks from `registration_video/via_masks_v4.npz` (latest). Generated from CSV polyline annotations, NOT from auto-thresholding images.

### V4 VIA annotations (CURRENT — all 21 tiles, 2026-03-27)
- User re-annotated rows 2-4 in VIA, consolidated CSV: `via_annotations/via_export_csv.csv`
- Images for VIA annotation: `via_annotations/row*_*_GFP_MIP.png` (4200×4200 each)
- row1_4 has **empty/unsupported annotation** — skip it (not in TILE_ORDER pipeline)
- row1_1 has 2 regions, all others have 1 region each
- Masks generated by `generate_masks_from_csv.py` using `cv2.fillPoly` on polyline points
- Coverage: 10-24% per tile (row1: 10-13%, row2: 14-20%, row3: 19-21%, row4: 20-24%, row5: 24%)
- Saved: `registration_video/via_masks_v4.npz` (keys: all 21 tiles)
- **IMPORTANT**: Always regenerate masks from CSV (not from image auto-thresholding)

### V4 QC outputs (2026-03-27)
- Annotated images on ALL GFP z-slices + MIP: `via_annotations/annotated_qc_new/{key}/GFP_*_annotated.png`
  - 21 tiles × 13 images = 273 annotated images
  - Green outline + semi-transparent fill = polyline region; red dot = start, blue dot = end
- Script: `draw_annotations_qc.py` — reads CSV, draws on all GFP images in `png_exports/registration_video/{key}/`

### Mask generation scripts
- `generate_masks_from_csv.py` — **CORRECT method**: reads `via_annotations/via_export_csv.csv`, fills polylines with `cv2.fillPoly`, saves to `via_masks_v4.npz`. Loads v2 masks as base and overwrites tiles present in CSV.
- `generate_masks_from_images.py` — **DO NOT USE**: auto-Otsu threshold from GFP MIP PNGs. User rejected this — masks were wrong.

- `auto_align_iou_v4.py` — 3-stage IOU search (coarse/medium/fine). Saves `auto_align_transforms_iou_v4.json`, QC to `png_exports/z_stitch_qc_iou_v4/`

### Z-stitch findings
- All tiles are (12, 2, 4200, 4200) uint16, DAPI+GFP, 0.645µm/px XY, 2µm z-step
- Tiles within a row are sequential z-stacks of the SAME FOV (not xy-tiled)
- GFP intensity increases monotonically row1→row5 (max: 4K→63K) confirming depth ordering
